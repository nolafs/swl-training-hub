# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security issues seriously. If you discover a security vulnerability, please follow these steps:

1. **Do not** open a public GitHub issue for security vulnerabilities
2. Email security concerns to [security@example.com] (replace with actual contact)
3. Include as much detail as possible:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### What to Expect

- **Acknowledgment**: We will acknowledge receipt within 48 hours
- **Assessment**: We will assess the vulnerability and determine its severity
- **Updates**: We will keep you informed of our progress
- **Resolution**: Once fixed, we will notify you and credit you (unless you prefer anonymity)

## Security Best Practices

When contributing to this project:

- Never commit secrets, API keys, or credentials
- Keep dependencies updated
- Follow secure coding practices
- Use environment variables for sensitive configuration
